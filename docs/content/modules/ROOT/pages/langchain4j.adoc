= LangChain4j Integration
:navtitle: LangChain4j

Vectors provides a LangChain4j `EmbeddingStore` adapter for using java-vectors as the storage backend in LangChain4j applications. `JavaVectorsEmbeddingStore` implements LangChain4j's `EmbeddingStore<TextSegment>`, backing it with an indexed, optionally persistent `VectorCollection`.

NOTE: Vectors requires JDK 25 with the Vector API enabled: run with `--add-modules jdk.incubator.vector`.

== Dependency Setup

[tabs]
====
Gradle::
+
[source,kotlin,subs="attributes+"]
----
dependencies {
    implementation("com.integrallis:vectors-langchain4j:{vectors-version}")
    implementation("dev.langchain4j:langchain4j-core:1.13.1")
}
----

Maven::
+
[source,xml,subs="attributes+"]
----
<dependency>
    <groupId>com.integrallis</groupId>
    <artifactId>vectors-langchain4j</artifactId>
    <version>{vectors-version}</version>
</dependency>
----
====

== Using JavaVectorsEmbeddingStore

Build a `VectorCollection` and pass it to `JavaVectorsEmbeddingStore.builder(collection)`. Unlike
the Spring AI adapter, LangChain4j supplies pre-computed embeddings, so the store needs no
`EmbeddingModel`:

[source,java]
----
VectorCollection collection = VectorCollection.builder()
    .dimension(384)
    .metric(SimilarityFunction.COSINE)
    .indexType(IndexType.HNSW)
    .build();

EmbeddingStore<TextSegment> store = JavaVectorsEmbeddingStore.builder(collection)
    .commitAfterAdd(true)   // commit after each add/addAll; default true
    .build();
----

=== Adding Embeddings

Embeddings are pre-computed by a LangChain4j `EmbeddingModel` and passed to the store:

[source,java]
----
EmbeddingModel model = new AllMiniLmL6V2EmbeddingModel();

TextSegment segment = TextSegment.from("hello world",
    Metadata.from("author", "alice"));
Embedding embedding = model.embed(segment).content();

String id = store.add(embedding, segment);
----

=== Searching

[source,java]
----
Embedding queryEmbedding = model.embed("hello").content();

List<EmbeddingMatch<TextSegment>> matches = store.search(
    EmbeddingSearchRequest.builder()
        .queryEmbedding(queryEmbedding)
        .maxResults(10)
        .build())
    .matches();
----

=== Deleting

[source,java]
----
import static dev.langchain4j.store.embedding.filter.MetadataFilterBuilder.metadataKey;

store.remove(id);                                          // by id
store.removeAll(List.of(id1, id2));                        // by id collection
store.removeAll();                                         // all documents
store.removeAll(metadataKey("author").isEqualTo("alice")); // by filter
----

== Metadata Mapping

LangChain4j `Metadata` values are converted to java-vectors `MetadataValue`:

[cols="1,1", options="header"]
|===
| LangChain4j type | MetadataValue type

| `String`  | `MetadataValue.StringValue`
| `Integer` / `Long` | `MetadataValue.LongValue`
| `Float` / `Double` | `MetadataValue.DoubleValue`
| `Boolean` | `MetadataValue.BooleanValue`
|===

== Filter Translation

LangChain4j filter predicates are translated to the java-vectors `Filter` AST.
All standard LangChain4j filter operators are supported.

== Embedding Cache

Combine with `vectors-cache-langchain4j` to cache embedding model calls. `CachingEmbeddingModel`
caches embedding vectors keyed by input text:

[source,java]
----
VectorCache<String, float[]> cache = CaffeineVectorCache.<String, float[]>builder()
    .maximumSize(10_000)
    .build();

EmbeddingModel cached = new CachingEmbeddingModel(embeddingModel, cache);
----

`CachingEmbeddingStore` wraps an `EmbeddingStore`, caching search results keyed by the
`EmbeddingSearchRequest`:

[source,java]
----
VectorCache<EmbeddingSearchRequest, EmbeddingSearchResult<TextSegment>> searchCache =
    CaffeineVectorCache.<EmbeddingSearchRequest, EmbeddingSearchResult<TextSegment>>builder()
        .maximumSize(10_000)
        .build();

EmbeddingStore<TextSegment> cachedStore = new CachingEmbeddingStore<>(store, searchCache);
----

Write-through semantics: mutations (`add`, `addAll`, `remove`, `removeAll`) invalidate the search
cache automatically.
