= Vectors

Embedded vector search and persistence for Java 25, built on the JDK Vector API (`jdk.incubator.vector`) for SIMD distance kernels with a scalar fallback.

Vectors runs in-process. There is no separate service to deploy, no network hop, and no serialization boundary between your application and the index. You create a collection, add documents, and search -- all inside the JVM.

== Overview

Vectors provides:

* **SIMD distance kernels** -- dot product, L2 (Euclidean), and cosine similarity vectorized with the Java Vector API, with a pure-scalar fallback for platforms without vector hardware.
* **Five CPU index types** -- FLAT (exact), HNSW, Vamana/DiskANN, IVF_FLAT, and IVF_PQ. (GPU-accelerated CUVS indexes are part of the separately-licensed DartVault tier.)
* **Vector quantization** -- scalar (SQ8/SQ4), product (PQ), binary (BQ/BBQ), RaBitQ, NVQ, and TurboQuant.
* **Durable persistence** -- memory-mapped storage with atomic, generation-based commits, recovery on restart, tombstone deletes, and background compaction.
* **Framework adapters** -- Spring AI `VectorStore`, LangChain4j `EmbeddingStore`, a Spring Boot starter, and semantic-cache modules.
* **Off-heap memory** -- vectors are stored via `MemorySegment` and `Arena` (Foreign Function & Memory API), so large collections do not pressure the garbage collector.

== Quick Start

Add `vectors`, the single umbrella dependency. It re-exports the embedded database facade (`vectors-db`) that unifies every index and quantizer behind a single API, so one dependency is enough for most applications:

[tabs]
====
Gradle::
+
[source,kotlin,subs="attributes+"]
----
dependencies {
    implementation("com.integrallis:vectors:{vectors-version}")
}
----

Maven::
+
[source,xml,subs="attributes+"]
----
<dependency>
    <groupId>com.integrallis</groupId>
    <artifactId>vectors</artifactId>
    <version>{vectors-version}</version>
</dependency>
----
====

Create a collection, add documents, and search:

[source,java]
----
try (VectorCollection collection = VectorCollection.builder()
        .dimension(384)
        .metric(SimilarityFunction.COSINE)
        .indexType(IndexType.HNSW)
        .build()) {

    collection.add("a", embedding1, "hello world")
              .add("b", embedding2, "goodbye world")
              .commit();

    SearchResult result = collection.search(
        SearchRequest.builder(queryEmbedding, 5).build());

    for (SearchResult.Hit hit : result.hits()) {
        System.out.println(hit.id() + " -> " + hit.score());
    }
}
----

NOTE: Requires JDK 25 with `--add-modules jdk.incubator.vector` at compile and runtime. See xref:getting-started.adoc[Getting Started] for build configuration.

== Modules

[cols="1,3"]
|===
|Module |Description

|`vectors-db`
|Embedded database facade: the `VectorCollection` API over every index and quantizer, with metadata filtering, persistence, deletion, and compaction. The entry point for most applications.

|`vectors-core`
|SIMD distance kernels, vector types, and similarity functions. Foundation module with no internal dependencies.

|`vectors-storage`
|Off-heap memory, mmap file I/O, and arena-scoped storage.

|`vectors-quantization`
|Scalar, product, binary, RaBitQ, NVQ, and TurboQuant quantizers.

|`vectors-hnsw`
|HNSW graph index.

|`vectors-vamana`
|Vamana/DiskANN graph index.

|`vectors-ivf`
|IVF family indexes (IVF_FLAT, IVF_PQ).

|`vectors-spring-ai`
|Spring AI `VectorStore` adapter and Spring Boot starter.

|`vectors-langchain4j`
|LangChain4j `EmbeddingStore` adapter.

|`vectors-bench`
|JMH microbenchmarks and an ANN-Benchmarks-compatible macrobenchmark harness.
|===

See xref:architecture.adoc[Architecture] for the full module graph and design.

== Requirements

* **JDK 25** with `--add-modules jdk.incubator.vector`. The Vector API is a hard requirement; SIMD distance kernels are the core of the library.
* **Gradle** (the wrapper is included) or **Maven 3.9+**.

== Roadmap

The xref:distributed.adoc[distributed tier] (`vectors-distributed`, `vectors-cluster`, `vectors-server`, `vectors-gpu`) extends the same engine into an object-storage-backed database that scales to billions of vectors at object-storage cost. The embedded library documented here is Apache 2.0.
